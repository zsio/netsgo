## 二、全部方向汇总表

### 🏗️ 核心基础建设（Phase 2 必做）

| # | 方向 | 描述 | 价值 | 难度 |
|---|------|------|------|:----:|
| 1 | **Agent 接入认证** | Key/Token 认证，防止未授权 Agent 连接；数据通道一次性 Token | 安全基础，不做无法上生产 | ⭐⭐ |
| 2 | **SQLite 持久化** | 代理规则、Agent 元数据、历史探针数据、审计日志的本地存储 | 重启不丢配置，数据可追溯 | ⭐⭐ |
| 3 | **Web 面板 CRUD** | Agent 列表/在线状态、隧道创建/编辑/删除、系统设置 | 告别命令行，零门槛使用 | ⭐⭐ |
| 4 | **探针监控大屏** | 接入 shadcn ui chart，绘制 CPU/内存/网络实时折线图 | 差异化特性，打动运维主管 | ⭐⭐ |
| 5 | **go:embed 嵌入前端** | 编译时将前端 dist 嵌入 Server 二进制，单文件交付 | 杀手级部署体验 | ⭐ |

### 🌐 核心隧道能力

| # | 方向 | 描述 | 价值 | 难度 |
|---|------|------|------|:----:|
| 6 | **HTTP(S) 泛域名路由** | `*.tunnel.example.com` 根据子域名自动路由到 Agent；内置 ACME 自动证书 | frp 用户最大痛点，做好就能抢用户 | ⭐⭐⭐ |
| 7 | **UDP 隧道** | 支持 UDP 协议转发，覆盖游戏/DNS/VoIP 场景 | 补全协议矩阵 | ⭐⭐⭐ |
| 8 | **P2P 打洞** | STUN/TURN 辅助直连，Server 只做信令协调，流量不经过 Server | 降低带宽成本，降低延迟 | ⭐⭐⭐⭐ |
| 9 | **QUIC 传输层** | 用 QUIC 替代 WebSocket+yamux，原生多路复用 + 0-RTT 重连 + 连接迁移 | 弱网/移动场景体验质变 | ⭐⭐⭐⭐ |
| 10 | **SSH 反向隧道兼容** | Server 内嵌 SSH Server，通过标准 `ssh -R` 即可建立隧道，无需安装 Agent | 零安装，受限环境唯一选择 | ⭐⭐⭐ |
| 11 | **eBPF 透明代理** | 通过 eBPF hook 内核网络栈，自动发现并转发所有监听端口，应用无感知 | 核弹级差异化，技术制高点 | ⭐⭐⭐⭐⭐ |
| 12 | **WireGuard 虚拟组网** | 所有 Agent 组成虚拟局域网，互相直接通信，对标 Tailscale/ZeroTier | 从穿透升级为组网 | ⭐⭐⭐⭐ |

### 🛡️ 安全与管控

| # | 方向 | 描述 | 价值 | 难度 |
|---|------|------|------|:----:|
| 13 | **ZTNA 零信任网关** | 每条隧道绑定访问控制策略（IP 白名单/OAuth/OTP），带身份验证的反向代理 | 把"工具"变成"安全产品"，对标 Cloudflare Tunnel | ⭐⭐⭐ |

| 15 | **定时端口快照** | Agent 定期扫描网段端口变化，发现新增/关闭的端口和未知设备 | 安全合规场景刚需（等保） | ⭐⭐⭐ |
| 16 | **智能限速 + QoS** | 动态流量策略：工作时间 SSH 优先，超配额降速不断开 | 公有服务的前提条件 | ⭐⭐ |

## 五、核心理念

> *内网穿透本质上是在解决"网络可达性"问题。一旦在用户的内网中有了一个 Agent，你就拥有了通往那个网络的钥匙。所有的展望方向都在回答一个问题：**拿到这把钥匙之后，还能为用户创造什么价值？***

> *做一个好用的内网穿透工具不难，难的是做一个让人离不开的网络基础设施。*
