# 已知问题

## ✅ P0 — 数据通道无认证（安全漏洞）

**状态**: **已修复** — `pkg/protocol/message.go`, `internal/server/data.go`, `internal/client/client.go`

**问题描述**:

当前数据通道握手仅发送 AgentID，而 AgentID 格式为 `agent_主机名_序号`，容易被猜测。攻击者可以直接向 Server 发送 `0x4E + AgentID` 来劫持 Agent 的数据通道。

**修复内容**:

1. `AuthResponse` 新增 `DataToken string` 字段（`crypto/rand` 生成 16 字节随机 token）
2. Server 在 `handleAuth()` 中生成 DataToken，存入 `server.dataTokens` sync.Map（token → agentID）
3. Client 握手改为发送 DataToken 而非 AgentID
4. Server 在 `handleDataConn()` 中用 `LoadAndDelete` 一次性验证 token（防重放）

## ✅ P0 — StartProxy TOCTOU 竞态条件

**状态**: **已修复** — `internal/server/proxy.go`

**问题描述**:

`StartProxy()` 中检查代理是否存在和写入之间释放了锁，并发请求可能绕过重名检查。

**修复内容**:

- 先用读锁做快速检查（快速失败）
- 调用 `net.Listen()` 绑定端口
- 再在写锁内做二次检查+插入；若发现重名则关闭已监听的端口并返回错误

## ✅ P1 — Agent ID 自增可预测

**状态**: **已修复** — `internal/server/server.go`

**问题描述**: `generateID()` 使用自增计数器，重启后归零产生重复 ID，且格式可预测。

**修复内容**: 改用 `crypto/rand` 生成 16 字节随机后缀；`generateID()` 重命名为 `generateRandSuffix()` 返回随机十六进制字符串。

---

## ✅ P1 — agent.Stats 读写无锁保护

**状态**: **已修复** — `internal/server/server.go`

**问题描述**: `controlLoop` 写 `agent.Stats` 和 `handleAPIAgents` 读 `agent.Stats` 在不同 goroutine 中无锁访问，存在 data race。

**修复内容**: 将 `Stats *protocol.SystemStats` 改为 `stats atomic.Pointer[protocol.SystemStats]`，提供 `GetStats()` / `SetStats()` 原子访问方法。

---

## ✅ P1 — PeekListener pending channel 满时阻塞

**状态**: **已修复** — `internal/server/server.go`

**问题描述**: `pending` channel 缓冲为 64，满时 `dispatchLoop` 阻塞，导致数据通道连接也无法被 accept。

**修复内容**:
- 缓冲区从 64 扩大到 1024
- HTTP 连接入队改为非阻塞 `select + default`，队满时关闭连接并打印警告日志

---

## ✅ P2 — requestProxy 使用硬编码 Sleep

**状态**: **已修复** — `internal/client/client.go`

**问题描述**: `requestProxy()` 用 `time.Sleep(500ms)` 等待数据通道就绪，不可靠。

**修复内容**: 添加 `dataReady chan struct{}` 信号通道；`waitForDataChannel(timeout)` 通过 `select` 等待 channel 关闭（即数据通道已建立），最多等待 3 秒后继续（附带警告日志）。

---

## ✅ P2 — Server 无 Graceful Shutdown

**状态**: **已修复** — `internal/server/server.go`, `cmd/server/main.go`

**问题描述**: `Server.Start()` 无关闭方法，进程只能通过 kill 退出。

**修复内容**:
- `Server` 结构体保存 `httpServer *http.Server` 引用
- 新增 `Stop()` 方法，使用 `context.WithTimeout(5s)` 调用 `http.Server.Shutdown()`
- `cmd/server/main.go` 监听 `SIGINT` / `SIGTERM` 信号，触发 `s.Stop()` 优雅退出
